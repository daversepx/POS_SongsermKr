# ระบบ POS ร้านเคมีเกษตร

เว็บแอป POS ครบชุด เก็บข้อมูลบนคลาวด์ (Supabase) ใช้ได้หลายเครื่องพร้อมกันในร้าน ซิงค์เรียลไทม์
รองรับ คิดเงิน · ปริ้นใบเสร็จ thermal · จัดการสต็อก · ระบบลูกค้า + ขายเชื่อ · รายงานขายรายวัน

---

## ภาพรวมสถาปัตยกรรม

```
  เครื่องหน้าร้าน ─┐
  เครื่องในออฟฟิศ ─┼──►  Supabase (ฐานข้อมูลกลางบนคลาวด์)
  มือถือเจ้าของ  ─┘        - PostgreSQL + Auth + Realtime
```

ทุกเครื่องเปิดเว็บเดียวกัน ล็อกอินแล้วเห็นข้อมูลชุดเดียวกัน
ขายของที่เครื่องหนึ่ง สต็อกที่เครื่องอื่นลดทันที

**หัวใจการออกแบบ:** ทุกการคุยกับฐานข้อมูลรวมอยู่ที่ `src/db.js` ไฟล์เดียว
อนาคตถ้าจะย้าย backend แก้แค่ไฟล์นี้ หน้าจอไม่ต้องแตะ

---

## โครงสร้างไฟล์

```
agri-pos/
├── index.html            หน้าหลัก + หน้าจอล็อกอิน
├── schema.sql            สคีมาฐานข้อมูล (รันใน Supabase ครั้งเดียว)
├── README.md             คู่มือนี้
└── src/
    ├── config.js         ★ ใส่ URL + key ของ Supabase ที่นี่
    ├── supabaseClient.js เชื่อมต่อ Supabase
    ├── auth.js           ล็อกอิน/ล็อกเอาท์
    ├── db.js             ★ ชั้นจัดการข้อมูลกลาง (ทุก query อยู่นี่)
    ├── app.js            ตัวควบคุม UI ทั้งหมด
    └── styles.css        สไตล์
```

## ติดตั้งทีละขั้น

### 1. สร้างฐานข้อมูล Supabase
1. สมัคร/เข้า https://supabase.com แล้วกด **New Project** (เลือก region Singapore จะเร็วสุดสำหรับไทย)
2. ตั้งรหัสผ่านฐานข้อมูลแล้วรอสร้างเสร็จ ~2 นาที
3. เข้าเมนู **SQL Editor** > **New query** > เปิดไฟล์ `schema.sql` คัดลอกทั้งหมดมาวาง > กด **Run**
   (ถ้าอยากได้สินค้าตัวอย่างทดสอบ ให้ลบเครื่องหมายคอมเมนต์ `--` ท้ายไฟล์ก่อนรัน)

### 2. สร้างบัญชีพนักงาน
1. เข้าเมนู **Authentication** > **Users** > **Add user**
2. ใส่อีเมล + รหัสผ่านของพนักงาน (อันนี้คือที่ใช้ล็อกอินเข้าแอป) สร้างได้หลายคน

### 3. เอาคีย์มาใส่ในแอป
1. เข้า **Project Settings** > **API**
2. คัดลอก **Project URL** และ **anon public key**
3. เปิด `src/config.js` แล้วแก้สองบรรทัดนี้:
   ```js
   export const SUPABASE_URL = "https://xxxx.supabase.co";
   export const SUPABASE_ANON_KEY = "eyJhbGci...";
   ```
   พร้อมแก้ชื่อร้าน/ที่อยู่/เบอร์ที่จะพิมพ์บนใบเสร็จในตัวแปร `SHOP`
   และใส่ **เบอร์พร้อมเพย์ของร้าน** ที่ `promptpayId` (ใช้สร้าง QR รับเงิน) เช่น `"0812345678"`
   หรือเลขบัตรประชาชนที่ผูกพร้อมเพย์ เช่น `"1234567890123"`

### 4. รันบนเครื่อง
ไฟล์นี้ใช้ ES Module จึงต้องเปิดผ่านเว็บเซิร์ฟเวอร์ (เปิดไฟล์ตรงๆ ไม่ได้)
เปิด terminal ในโฟลเดอร์ `agri-pos` แล้วใช้คำสั่งใดก็ได้ที่มี:

```bash
# ถ้ามี Python
python3 -m http.server 5173

# หรือถ้ามี Node
npx serve .
```

แล้วเปิดเบราว์เซอร์ไปที่ `http://localhost:5173` ล็อกอินด้วยบัญชีพนักงานที่สร้างไว้

### 5. ตั้งเครื่องปริ้นใบเสร็จ thermal
- ติดตั้งไดรเวอร์เครื่องปริ้น thermal แล้วตั้งเป็น **เครื่องปริ้นเริ่มต้น** ของเครื่อง
- ใบเสร็จถูกจัดขนาดไว้สำหรับ **80mm** แล้ว ถ้าใช้ **58mm** แก้ใน `src/styles.css`
  ที่ `@page{size:80mm auto;}` และ `.rcpt{width:80mm;}` ให้เป็น `58mm`
- เวลากดยืนยันการขาย เบราว์เซอร์จะเปิดหน้าพิมพ์ให้อัตโนมัติ

## เปิดให้หลายเครื่องใช้ (Deploy)

วิธีง่ายสุด ฟรี และเหมาะกับร้านเดียว:

1. สมัคร https://www.netlify.com (หรือ Vercel / Cloudflare Pages)
2. ลากโฟลเดอร์ `agri-pos` ทั้งโฟลเดอร์ไปวางในหน้า **Deploy** (drag & drop)
3. จะได้ลิงก์เช่น `https://ร้านคุณ.netlify.app`

### เพิ่มเครื่องที่สอง สาม...
แค่เปิดลิงก์เดียวกันบนเครื่องนั้น แล้วล็อกอิน — **เห็นข้อมูลชุดเดียวกันทันที ไม่ต้องตั้งค่าอะไรเพิ่ม**
นี่คือข้อดีของการวางบนคลาวด์ตั้งแต่แรก: ขยายจากเครื่องเดียวเป็นหลายเครื่องโดยไม่ต้องย้ายข้อมูลเลย

## สิ่งที่ทำได้แล้ว
- คิดเงิน 4 วิธี: **เงินสด · พร้อมเพย์ QR (สร้าง QR ตามยอดจริง) · บัตรสินเชื่อเกษตรกร (ธ.ก.ส.) · ขายเชื่อ**
  (สามแบบแรกร้านได้เงินครบทันที มีแค่ "ขายเชื่อ" ที่ลงยอดค้างลูกค้า)
- ตัดสต็อกอัตโนมัติแบบ atomic (กันสต็อกติดลบเมื่อขายพร้อมกันหลายเครื่อง)
- ค้นหาสินค้าด้วยชื่อ/รหัส (รองรับ 200+ SKU)
- **พักตะกร้า / เรียกคืนตะกร้า** สำหรับขายลูกค้าหลายรายสลับกันในเครื่องเดียว
- **ลงต้นทุนสินค้า** + รายงาน **กำไรขั้นต้นรายวัน** (เก็บต้นทุน ณ เวลาที่ขายจริง)
- **ขายแบบจ่ายก่อน-รับทีหลัง (รอรับสินค้า)**: เลือก "รับภายหลัง" ตอนคิดเงิน → กันของให้ลูกค้า → ทยอยมารับหลายครั้งได้ พิมพ์ **ใบส่งมอบสินค้า** ทุกครั้ง
- **ยกเลิกบิล**: คืนสต็อกอัตโนมัติ + ตัดออกจากยอดขาย/กำไร/ยอดค้าง
- **ค้นหา/พิมพ์ซ้ำใบเสร็จย้อนหลัง** (ตามเลขที่บิล/ชื่อลูกค้า/วันที่)
- เอกสาร: **ใบเสร็จรับเงิน** (แบบทางการ) และ **ใบส่งมอบสินค้า** พิมพ์เครื่อง thermal 80mm ได้
- ระบบลูกค้า: ประวัติการซื้อ ยอดค้างชำระ บันทึกรับชำระหนี้
- รายงานรายวัน: แยกยอดตามวิธีจ่ายเงิน สินค้าขายดี และยอดค้างชำระทั้งร้าน
- ล็อกอินพนักงาน + บันทึกว่าใครขายบิลไหน · ซิงค์เรียลไทม์ทุกเครื่อง

> **ฐานข้อมูลเดิมที่ติดตั้งไปก่อนแล้ว** ให้รัน `migration-upgrade.sql` เพิ่มเติมเพื่อรองรับฟีเจอร์ ต้นทุน/รอรับสินค้า/ยกเลิกบิล (ฐานข้อมูลใหม่รัน `schema.sql` ไฟล์เดียวครบ)

## ต่อยอดได้อีก (ยังไม่รวม)
- สแกนบาร์โค้ดด้วยเครื่องอ่าน
- นำเข้า 200 SKU จากไฟล์ Excel ทีเดียว
- ออกใบกำกับภาษี / รายงานสรุปรายเดือน
- โหมดออฟไลน์กันเน็ตหลุด (cache + sync ทีหลัง)
- แจ้งเตือนสินค้าใกล้หมด / สารเคมีใกล้หมดอายุ

## หมายเหตุ
- anon key เป็นคีย์สาธารณะ ปลอดภัยที่จะอยู่ฝั่งหน้าบ้าน เพราะข้อมูลถูกป้องกันด้วย Row Level Security (กำหนดให้เฉพาะผู้ล็อกอินเข้าถึงได้) อย่าเอา **service_role key** มาใส่เด็ดขาด
- สินค้าตัวอย่างในไฟล์ schema ใส่เฉพาะสารที่ยังขายได้ตามกฎหมาย ตอนใช้จริงใส่รายการของร้านเองได้เลย
